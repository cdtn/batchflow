Base Inception¶

Parent class to inception models

class Inception(*args, **kwargs)[source]¶

Bases: batchflow.models.tf.base.TFModel

The base class for all inception models

Configuration

bodydict
layoutstr

a sequence of blocks in the network:

  • b - inception block for v1 and v3 models

  • r - reduction block

  • f - factorization_block for Inception_v3 model (see factorization_block())

  • m - mixed_block for Inception_v3 model (see mixed_block())

  • e - expanded_block for Inception_v3 model (see expanded_block())

  • A - inception block A for Inception_v4 model (see inception_a_block())

  • B - inception block B for Inception_v4 model (see inception_b_block())

  • C - Inception block C for Inception_v4 model (see inception_c_block())

  • G - grid-reduction block for Inception_v4 model (see reduction_grid_block())

archdict

parameters for each block:

keystr

block’s short name

valuedict

specific parameters (e.g. filters)

classmethod block(inputs, filters, layout='cn', name=None, **kwargs)[source]¶

Inception building block

Parameters

  • inputs (tf.Tensor) – input tensor

  • filters (list with 6 items:) –

    • number of filters in 1x1 conv

    • number of filters in 1x1 conv going before conv 3x3

    • number of filters in 3x3 conv

    • number of filters in 1x1 conv going before conv 5x5,

    • number of filters in 5x5 conv,

    • number of filters in 1x1 conv going before max-pooling

  • layout (str) – a sequence of layers in the block. Default is ‘cn’.

  • name (str) – scope name
